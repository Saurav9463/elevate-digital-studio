Build a password-protected Admin Dashboard for a web studio website called Elevate Web Solutions. The dashboard allows the site owner to edit all content that appears on the public website. All data is stored in Supabase (use the Supabase client). The public website reads content from Supabase instead of the hardcoded src/data/site.ts file.

Auth

Simple login page with email + password using Supabase Auth
Route: /admin — redirect to /admin/login if not authenticated
After login, go to /admin/dashboard


Dashboard Sidebar Navigation
Sections: Hero, Trust Stats, Services, Projects, Testimonials, Founders, Site Config, Process, Why Choose Us

Section: Hero
Editable fields:

eyebrow (text input)
title (text input)
description (textarea)


Section: Trust Stats
A list of 4 stats, each with:

value (e.g. "40+")
label (e.g. "Projects delivered")

Ability to edit each stat inline.


Section: Services
A table/list of services. Each service has:

title, summary, bullets (comma-separated), icon (text), slug

Ability to add, edit, and delete services.


Section: Projects
Each project has:

title, category, client, summary, year
tech (comma-separated tags)
live_url, cover_url
is_featured (toggle)
gradient (text input)

Ability to add, edit, delete, and reorder projects. Mark up to 3 as featured.


Section: Testimonials
Each testimonial has:

name, role, company, business, project
quote (textarea)
rating (1–5 star selector)
avatar_url

Ability to add, edit, delete.


Section: Founders
Each founder has:

name, role, bio (textarea)
photo_url
skills (comma-separated)
linkedin_url, github_url

Ability to edit (2 founders, no add/delete needed).


Section: Site Config
Editable fields:

name, tagline, description
email, whatsapp, whatsappUrl
linkedin, github, location
formspreeEndpoint


Section: Process Steps
4 steps each with step (e.g. "01"), title, text. Editable inline.

Section: Why Choose Us
4 items each with title and text. Editable inline.

UI Requirements

Dark sidebar, clean white/light content area
Use shadcn/ui components (already installed: Input, Textarea, Button, Table, Dialog, Switch, Badge, Card, Tabs, Toast)
Use Tailwind CSS
Each section has a Save Changes button that writes to Supabase
Show a success toast on save
Show a loading state while fetching/saving
Responsive — works on desktop and tablet


Supabase Tables to Create
Create these tables in Supabase with matching column names as the fields above:

site_config, hero, trust_stats, services, projects, testimonials, founders, process_steps, why_choose_us

Tech Stack

React + TypeScript + Vite (TanStack Router for routing)
Supabase JS client (@supabase/supabase-js)
shadcn/ui + Tailwind CSS
All env vars via VITE_SUPABASE_URL and VITE_SUPABASE_ANON_KEY
